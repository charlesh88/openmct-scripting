Open MCT Views Generator v3.0

CSV as a source

What it does

  • For each telemetry point in the input file, a Condition Set with a default up to 4 conditions that allow all the standard Open MCT value comparisons. Condition parameters and values are set in the input CSV file.
  • For each Condition Set, a Condition Widget that uses the Condition Set for dynamic styling, based on colors and label text defined in the input file.
  • A Display Layout that includes all the widgets in a grid layout defined by the settings below.
  • A Display Layout that includes all the telemetry points in the input file as alphanumerics, in a grid layout defined by the settings below. Each alpha has a text label from the "name" column in the input file.
  • A LAD Table that includes all the telemetry points in the input file.
  • Folders that hold the Condition Sets and Widgets.
  • A single folder that holds all of the above, named using your input Root Name below.

How to use

  1. Make a CSV file like this example. Note that it must have a header row.
  2. Set parameters below. The smaller the Layout Grid numbers are the closer you can place items with a margin, but width and heights will need to be adjusted accordingly. Using "Columns" will build across first, "Rows" builds down first. For example, with a total of 16 items, using "Columns" set to 2 will result in two columns of elements with 8 rows each.
  3. Press the Telemetry "Choose File" button and select your telemetry CSV file.
  4. Optionally, press the Matrix Layout "Choose File" button and select a matrix layout CSV file.
  5. Press "DOWNLOAD" to download the generated JSON file.
  6. Import the generated file into your Open MCT environment using the "Import from JSON" action.

Pride procedure .prl files as a source

What it does

  • Each selected .prl file will be traversed, and steps with telemetry in them as data references will be extracted. Steps without telemetry will be ignored.
  • For each procedure, a Display Layout will be genned that includes all its telemetry points as alphanumerics. Each alphanumeric will have a text label, based on the telemetry path. Labels will also be added for procedure step numbers.
  • For each procedure, a Stacked Plot will be genned that includes all its telemetry points.
  • A single Tabs view will be created that includes all the generated Display Layouts.
  • Folders that hold the layouts and plots.
  • A single folder that holds all of the above, named using your input Root Name below.

How to use

  1. Set parameters below. Label Width is recommended to be 220.
  2. Press the "Choose File" button and select one or more Pride .prl files. A JSON file will immediately be generated.
  3. Press "DOWNLOAD" to download the generated JSON file.
  4. Import the generated file into your Open MCT environment using the "Import from JSON" action.
