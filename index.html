What this does

  • For each telemetry point in the input file, a Condition Set with a default up to 4 conditions that allow all the standard Open MCT value comparisons. Condition parameters and values are set in the input CSV file.
  • For each Condition Set, a Condition Widget that uses the Condition Set for dynamic styling, based on colors and label text defined in the input file.
  • A Display Layout that includes all the widgets in a grid layout defined by the settings below.
  • A Display Layout that includes all the telemetry points in the input file as alphanumerics, in a grid layout defined by the settings below. Each alpha has a text label from the "name" column in the input file.
  • A LAD Table that includes all the telemetry points in the input file.
  • Folders that hold the Condition Sets and Widgets.
  • A single folder that holds all of the above, named using your input Root Name.

How to use

  1. Make a CSV file like this example. Note that it must have a header row.
  2. TODO: Content here and example for the matrix layout CSV file.
  3. Set parameters at left. The smaller the Layout Grid numbers are the closer you can place items with a margin, but width and heights will need to be adjusted accordingly. Using "Columns" will build across first, "Rows" builds down first. For example, with a total of 16 items, using "Columns" set to 2 will result in two columns of elements with 8 rows each.
  4. Press the Telemetry "Choose File" button and select your telemetry CSV file.
  5. Optionally, press the Matrix Layout "Choose File" button and select a matrix layout CSV file.
  6. Press "DOWNLOAD" to download the generated JSON file.
  7. Import the generated file into your Open MCT environment using the "Import from JSON" action.
