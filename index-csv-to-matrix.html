What this does

  • For each parameter in the Telemetry input file, a Condition Set with a default of up to 4 conditions that allow all the standard Open MCT value comparisons. Condition parameters and values are set in the input CSV file.
  • A Display Layout based on the Matrix Layout CSV. This CSV can set telemetry to be displayed as an alphanumeric or a Condition Widget; each will use Conditional Styling as defined in the Telemetry input CSV.
  • Folders that hold the Condition Sets and Widgets.
  • A single folder that holds all of the above, named using the input Root Name.

How to use

  1. Make a Telemetry input file CSV file like this example. Note that it must have a header row.
  2. TODO: Content here and example for the matrix layout CSV file.
  3. Set parameters. The smaller the Layout Grid numbers are the closer you can place items with a margin, but width and heights will need to be adjusted accordingly. Using "Columns" will build across first, "Rows" builds down first. For example, with a total of 16 items, using "Columns" set to 2 will result in two columns of elements with 8 rows each.
  4. Press the Telemetry "Choose File" button and select your telemetry CSV file. The file will be loaded and processed.
  5. Press the Matrix Layout "Choose File" button and select a matrix layout CSV file. You must choose a Telemetry input first.
  6. Once both files have been loaded, press "DOWNLOAD" to download the generated JSON file.
  7. Import the generated file into your Open MCT environment using the "Import from JSON" action.
