What this does

  • Crawls selected GCS .py files and finds telemetry in getval and checkval calls.
  • Crawls selected Pride .prl files and finds telemetry in DataReferences and DataNomenclatures.
  • Outputs a .csv file table of de-duped unique telemetry paths with usage indication per procedure or GCS. If a Yamcs server is available, validation results will also be indicated.
  • To enable mission database validation, enter a valid Yamcs "parameters" web api URL in the input. A proxy server may be necessary when running locally due to CORS issues.
